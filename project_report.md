# Quantum-Secured Chat Application: Project Report

## 1. Problem Statement
In modern cryptography, the security of key exchange protocols (such as RSA or Diffie-Hellman) relies on the computational difficulty of certain mathematical problems like integer factorization or discrete logarithms. However, with the advent of quantum computing, algorithms like Shor's algorithm threaten to break these protocols, rendering classically established secure communications vulnerable. 

Furthermore, classical key generators rely on pseudo-random number generators (PRNGs) with initial seeds. If an attacker discovers the seed, they can completely compromise the key stream. There is a need for a secure communication system that generates inherently unpredictable keys and detects eavesdropping attempts automatically based on the laws of physics rather than computational complexity.

## 2. The BB84 Protocol
This project addresses the problem by implementing the BB84 protocol for Quantum Key Distribution (QKD). In BB84, quantum bits (qubits) are used to establish a shared secret key.
1. **Preparation**: Alice generates random bits and encodes them into qubits using two possible bases: rectilinear (standard |0⟩, |1⟩) and diagonal (|+⟩, |−⟩).
2. **Transmission**: The qubits are sent to Bob over a quantum channel.
3. **Measurement**: Bob receives the qubits and measures them in his own randomly chosen bases. 
4. **Sifting**: Alice and Bob communicate over a classical public channel to compare their chosen bases. They keep only the bits where their bases matched, discarding the rest. This forms the sifted key.
5. **Security Check**: According to the no-cloning theorem and measurement collapse in quantum mechanics, an eavesdropper (Eve) who intercepts and measures the qubits cannot do so without irreversibly altering their quantum state. This introduces a detectable error rate (Quantum Bit Error Rate, or QBER). By comparing a small portion of the sifted key over the public channel, Alice and Bob can detect Eve's presence if the QBER exceeds a theoretical threshold of ~11%.

## 3. System Design
The application is designed in a modular structure to handle quantum key exchange and subsequent AES encryption:
- **`bb84.protocol`**: Simulates the generation, measurement, and sifting of qubits using IBM's Qiskit. It acts as the key exchange handshake.
- **`crypto.aes_encryption`**: Utilizes HMAC-based Key Derivation Function (HKDF) to derive a 256-bit Advanced Encryption Standard (AES) key from the raw quantum key. It performs AES-GCM authenticated encryption/decryption for the chat messages.
- **`attacks.eve`**: Defines a Man-In-The-Middle (MITM) agent that intercepts qubits, measures them, and re-encodes them, effectively simulating an eavesdropping attack.
- **`chat.server` & `chat.client` / `chat.gui`**: Manages real-time socket connections. The connection initialization automatically triggers the BB84 protocol. The GUI provides a visual layout using Tkinter.
- **`viz.dashboard`**: A matplotlib-based visualization dashboard that illustrates the security characteristics of the key exchange.

## 4. Attack Demo Results
The MITM simulation effectively demonstrates the security guarantees of QKD. 
- In a **normal, undisturbed session**, the measurement mismatch is solely dependent on basic alignment capabilities, yielding a 0% QBER on the simulation platform, resulting in successful key generation.
- When **Eve is activated**, she intercepts and measures the qubits before forwarding them. Because she randomly guesses the measurement bases, she introduces incorrect states into the transmission. The calculated QBER systematically spikes to approximately 25%.
- **Detection**: The system detects the QBER surpassing the 11% safety threshold, halts the key establishment, aborts the session, and triggers a critical warning, successfully thwarting the MITM attack.

## 5. Comparison with Classical Systems
A classical MITM attack involves intercepting a communication channel, reading plain data, and transparently forwarding it. The communicating parties remain completely unaware of the breach. In contrast, the quantum system immediately flags the intrusion due to the fundamental physical properties of qubits collapsing upon measurement.

Additionally, this project compares classical keys (generated from a seeded PRNG) against quantum-derived keys. The dashboard visualizes the lack of inherent entropy in a seeded PRNG stream compared to the uniform distribution and low autocorrelation of the true random outcomes generated by quantum measurement. The quantum approach eliminates vulnerabilities tied to predictable seeds, delivering robust, future-proof security.
